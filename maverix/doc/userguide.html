Overview

Mail Verification Exchange, Maverix, is an enterprise class appliance that provides a secure anti-spam/anti-virus solution.  The solution eliminates spam by interrogating each email that arrives.  Emails from known spammers or spam domains are immediately deleted.  Emails from known senders or domains are immediately delivered to the user’s inbox. However, emails from unknown senders are challenged to one-time-verify as a valid Internet sender intending to send valid email.  This is accomplished using our patent pending technology called Email Handshake Verification.  All emails from unknown senders are held in a quarantine database until either the sender is verified by the user, self-verified by the sender, or ultimately is deleted after 1,2 or 3 weeks (based on the user’s preferences).

See diagram 1 below.  The sender self-verification option sends a welcome message to the sender of an incoming email requesting that the sender click on a web link to become a valid sender within the user’s whitelist.  However, if the self-verification message bounces due to an “invalid user” error, as they often do for spammers that spoof their origin using invalid address, the sender is automatically be blocked.  All of these functions combine to greatly reduce the need for users to micro-manage their anti-spam service or worry about lost email.

With Maverix, you can:

  • Stop spam before it reaches your desktop therefore reducing you Deleted folders and speeding up your email client performance
  • Prevent web-based HTML spam messages from generating pop-ups, displaying offensive material, or try to download destructive software
  • Eliminate viruses before they reach your computer
  • Prevent spam with embedded tracking ids that notify a spammer of the validity of your email address
  • Guarantee immediate delivery of all your valid email from desired senders
  • Allow senders to verify themselves so that their email is immediately delivered
  • Directly manage your personal settings including vacation, working hours, digest frequency, and other options
  • Maintain subscriptions to both external and internal mailing lists
  • Send mail within the company as well as to major everyday clients without anti-spam checking

User Verification Process

All incoming email is interrogated to determine if the sender or the sender domain is already on the user’s whitelist per Diagram 1.  If the sender is whitelisted, then the email is immediately delivered to the user’s mail system.  If the system is blacklisted, then the email is dropped.  If the sender is not whitelisted or blacklisted, then the message is automatically greylisted and quarantined while Maverix’s verification process follows.


Diagram 1

If the user has sender self-verification enabled, then the sender will receive a verification handshake message from the Maverix system.  If the sender clicks on the “Verify” like within the message, then the sender will be added to the user’s whitelist.  Otherwise, the user will periodically receive a digest message that lists all email within the quarantine database.  The user will then select which senders to add to his personal whitelist, to his blacklist, or the user can simply forward or drop that specific message.

The Maverix system uses an internal email address such as maverix@maverix.domain.com when sending.  Using this sender name helps prevent verification message loops between two verification systems as well as clearly identifies the system to the recipient.

Maverix also uses a standard subject header when sending to a user or sender:

  Maverix Anti-Spam: 1 Message(s) pending verification

The body of the Maverix quarantine digest message contains clear instructions on how to manage the pending messages.

The user quarantine digest displays a summary of the emails pending within the quarantine database.  The user then has the options to take the following actions on each message:

  • Permit - Allow this sender to always send email by adding the sender to the user’s whitelist

  • Block -     Deny all email from this sender by adding this sender to the user’s blacklist

  • Forward     - Forward only this message and the sender remains on the greylist

  • Drop     - Drop this particular message and the sender remains on the greylist

When the user clicks on the first action, a new browser window will launch and the user will see a complete listing of all quarantined email awaiting the user’s actions. 

The user can either perform an action on each individual message or can check the boxes to the left and perform an aggregate operation:

  • Permit Checked – All senders with the box checked are added to the user’s whitelist
  • Block Checked - All senders with the box checked are added to the user’s blacklist
    Forward Checked     - All messages with the box checked are delivered to the user and the sender remains on the greylist
  • Drop Checked - All messages with the box checked are delivered to the user and the sender remains on the greylist

Clicking on a link may sometimes produce the following error:

Sorry, that verification request in invalid, Please check for a more recent verification URL

Maverix uses a complex algorithm to generate its secure one-time, short-lived URLs used for managing email delivery.  Periodically, the Maverix system will send a digest to the user and will generate all new URLs used for the aforementioned actions.  When a new digest is generated, the old URLs are expired and the new URLs are active.  Because Maverix does not require for users to remember more and more passwords, all security is managed using short-lived URLs embedded within the email messages.  The digest messages are delivered directly to the user who must use a secure email connection and password to receive the digest emails.  In fact, Maverix is password protected, not via additional passwords, but rather using the existing one for regular email. 

Often, spammers send spam with unique sender addresses per spam, like perf-message.16051754.0.0.4@somwhere.com.  It is obvious that this spammer uses some sort of auto-generating sender id for tracking the performance of the spamming campaign.  Since it will be unlikely to see this sender id again, dropping the particular message from the Maverix server will have the same effect as blocking the sender.

Dropping spam on the Maverix server instead of within the client software provides a variety of advantages.  Many spammers use a brute force dictionary attack against a corporate mail gateway and embed tracking objects within HTML messages to determine whether an email address is valid or not.  Since Maverix drops the email at the server, spammers are unable to track the success of their campaigns.  Other benefits of dropping unwanted email on the server include prevention of viruses, pop-ups, security vulnerabilities within the email client, and less consumption of system resources

Sender Verification Process

Sender verification is a highly valuable option that allows for new senders to immediately deliver their email to intended users.  Upon sending an email to a Maverix user, the sender will immediately receive a challenge handshake message to ensure that the sender is in fact a real user on the Internet intending to send a message.  Below is an example, of a typical Maverix challenge handshake message:

 

You have recently sent an email message to address specified below. In order to have your message delivered, click on Verify link after each email address:

  • joeuser@somecompany.com Verify

Once you have verified, your E-mail will be allowed to pass through and you will be put on list of verified addresses and you will not have to go through this process again.

 

Often a spammer uses a spoofed email address and therefore the challenge message bounces due to an “invalid user” error.  Maverix will automatically drop any challenge messages that bounce from an explicit invalid user error message.  This does not mean that if a sender’s emails server is down, then email will get dropped.  A message is only dropped if the bounce contains an invalid user error message which is distinctively different from other errors.  Email should be sent with valid return sender email addresses.  If there is an invalid email address that should directly deliver email to a user, then the user can enter that address within the user preferences interface.  

User Preference Parameters

In the top right hand corner of each user digest browser, opened after clicking on the first action, is a link to the user’s individual account settings.

Spam Blocked

The Spam Blocked value is a counter that tracks all spam either auto-blocked via bounced challenges or blocked via the user blacklist or user drop.

There should be five TABs listed in the top right corner:

  • Messages - Displays all actionable messages pending within the quarantine database.  User can then Permit, Block, Forward, Drop email messages.
  • Properties - User interface for customizing email parameters
  • Whitelist – Shows all senders and domains allowed to directly send mail to users
  • Blacklist - Shows all senders and domains that are denied from sending email
  • Greylist - Shows all senders that are pending verification

Messages

The Messages TAB is displayed by default and should show all messages within the quarantine database pending verification.  These messages are displayed in the standard digest format described in the User Verification section.

Properties

The Properties TAB maintains all the user specific configuration parameters.

Maverix Account Type:

  • Put unknown email on hold for verification - All unknown sender email will get stored within the quarantine database pending verification.  If sender self-verification is enabled, then they will receive challenge messages
  • I want all email pass through directly to my mailbox - All email for the user passes directly through the system unchecked.  This is typically the Default mode for installation.
  • Digest with unverified messages/senders will be sent every - Periodic interval that the digest message will be sent to the user.  A good default value is every two hours.  Maverix will not blindly fill up the user’s mailbox with digest messages but will instead intelligently handle digest management.
  • Digest will be sent within this period of day - This is the “working day” start and stop time that the digest messages will be delivered.  This is so that Maverix doesn’t continue to send digests after hours.  The “From” time is that start of the day at which digests begin and the “To” is the time at which they stop.

Sender Self-Verification:

  • Yes, Allow unknown senders to perform self-verification - New senders will receive a challenge handshake message and bounces will get auto-dropped.
  • No, I will verify unknown senders by myself - New senders will not receive challenge handshake messages and instead all new senders must be verified by the user.

After completing any change, select the Update button to save any changes.

Whitelist

All senders permitted to deliver email directly to a user are listed within the Whitelist TAB.  Verified sender email addresses are listed in alphabetical order.  You can change the action by selecting the “Block”, “Pend”, or “Delete” buttons.

  • Block – Deny email from this sender or domain.  Places sender or domain on blacklist
  • Pend – Require sender to perform verification.  Places sender or domain on greylist
  • Delete – Delete sender or domain from whitelist.  Removes sender or domain from list

New senders or domains can be added to the list by simply typing them in the box “Add new Email/Domain”.  Currently Maverix supports the following:

  • Adding a full sender name i.e. joe@somewhere.com
  • Adding a full domain name i.e. mail.host.domain.name.com
  • Adding the top two levels of a domain name: i.e. spammer.com, bogusmail.com

Blacklist

All senders explicitly blocked from sending email to a user are listed within the Blacklist TAB.  Blocked sender email addresses are listed in alphabetical order.  You can change the action by selecting the “Permit”, “Pend”, or “Delete” buttons.

  • Permit – Permit email from this sender or domain.  Places sender or domain on whitelist.
  • Pend – Require sender or domain to perform verification.  Places sender or domain on greylist
  • Delete – Delete sender or domain from blacklist.  Removes sender or domain from list. 

New senders or domains can be added to the list by simply typing them in the box “Add new Email/Domain”.  Currently Maverix supports the following:

  • Adding a full sender name i.e. joe@somewhere.com
  • Adding a full domain name i.e. mail.host.domain.name.com
  • Adding the top two levels of a domain name: i.e. spammer.com, bogusmail.com

Greylist

All senders requiring verification when sending email to a user are listed within the Greylist TAB.  Unverified sender email addresses are listed in alphabetical order.  When a message is either forwarded or dropped, the sender remains in an unverified state and resides on the greylist until either moved to the blacklist or whitelist.  You can change the action by selecting the “Permit”, “Block”, or “Delete” buttons.

  • Permit – Permit email from this sender or domain.  Places sender or domain on whitelist.
  • Block – Deny email from this sender or domain.  Places sender or domain on blacklist.
  • Delete – Delete sender or domain from greylist.  Removes sender or domain from list.

New senders or domains can be added to the list by simply typing them in the box “Add new Email/Domain”.  Currently Maverix supports the following:

  • Adding a full sender name i.e. joe@somewhere.com
  • Adding a full domain name i.e. mail.host.domain.name.com
  • Adding the top two levels of a domain name: i.e. spammer.com, bogusmail.com
